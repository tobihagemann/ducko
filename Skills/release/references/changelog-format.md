# Changelog Format Reference

## Format Rules

- Each entry starts with `- ` (dash space)
- Use past tense consistently ("Added", "Fixed", "Improved", "Removed")
- End each line with a period
- Reference GitHub issue/PR numbers with `(#NNN)` at the end when applicable

## Style Guide

- Focus on **user outcomes**, not technical implementation — describe how the change benefits the user, not what was changed internally
- Front-load the verb so users scanning the list quickly understand what changed
- Use active voice — avoid passive constructions
- Avoid "now" (e.g., "Vaults load faster" not "Vaults now load faster")
- Avoid third-person "we" language (e.g., "Added support for..." not "We added support for...")
- Keep entries concise — one line per change
- Prioritize entries by impact: headline features first, then improvements, then bug fixes
